## 🏗️ Архитектура приложения

### Основные компоненты

| Компонент | Описание | Технологии |
|-----------|----------|------------|
| **App.vue** | Главный контейнер без хедера (роутинг через `RouterView`) | Vue 3, Vue Router |
| **EditorPage.vue** | Страница редактора с верхней панелью и авторизацией | Vue 3, CodeMirror 6, Supabase Auth |
| **ExplorePage.vue** | Галерея скетчей сообщества с поиском и фильтрами | Vue 3, Supabase, Mock API |
| **SharePage.vue** | Страница публикации скетча | Vue 3, Supabase |
| **SketchDetailPage.vue** | Детальный просмотр скетча | Vue 3, Supabase |
| **CodeEditor.vue** | Редактор кода с подсветкой синтаксиса | CodeMirror 6, vue-codemirror6 |
| **P5Canvas.vue** | Изолированный iframe для выполнения p5.js скетчей | Iframe sandbox, Blob URL |
| **ConsoleOutput.vue** | Консоль для вывода сообщений и ошибок | Vue 3 |
| **ExamplesPanel.vue** | Галерея примеров с пошаговыми уроками | Vue 3, marked (markdown) |
| **AIChat.vue** | Чат с Deepseek AI для помощи с кодом | Deepseek API, Vue 3 |
| **AuthModal.vue** | Модальное окно входа/регистрации/сброса пароля | Vue 3, Supabase Auth |
| **UserProfile.vue** | Профиль пользователя с аватаром и настройками | Vue 3, Supabase Auth, Supabase Storage |

### Маршруты (Vue Router)

| Путь | Страница | Описание |
|------|----------|----------|
| `/` | EditorPage | Редактор кода (основная страница) |
| `/explore` | ExplorePage | Галерея скетчей сообщества |
| `/sketch/:id` | SketchDetailPage | Детали скетча |
| `/share` | SharePage | Публикация скетча |

### Поток данных

```
Router (Vue Router)
    │
    ├── EditorPage.vue
    │   ├── CodeEditor.vue ←→ code (v-model)
    │   ├── P5Canvas.vue → addMessage(), mouseMove
    │   ├── ConsoleOutput.vue → messages
    │   ├── ExamplesPanel.vue → loadExample(code)
    │   └── AIChat.vue → suggestCode(code)
    │
    ├── ExplorePage.vue
    │   └── mock/sketches.ts → search, filter, sort
    │
    ├── SharePage.vue
    │   └── Форма публикации → mock/real API
    │
    └── SketchDetailPage.vue
        └── mock/sketches.ts → getSketchById()
```

### Управление состоянием

- **Локальное состояние:** Каждый компонент управляет своим состоянием через `ref()`, `reactive()`
- **Двустороннее связывание:** `v-model` для редактора кода
- **События:** `emit()` для коммуникации между компонентами
- **localStorage:** Сохранение состояния в ExamplesPanel (выбранный пример, шаг, вид)
- **Mock API:** Функции в `src/mock/sketches.ts` для работы со скетчами сообщества

### Схема базы данных

**Таблицы:**

| Таблица | Описание |
|---------|----------|
| `profiles` | Профили пользователей (email, display_name, avatar_url, bio, role) |

**Роли пользователей:**
- `user` — обычный пользователь
- `moderator` — модератор (может одобрять/отклонять)
- `admin` — администратор (полный доступ)

### Безопасность и RLS (Row Level Security)

**Политики доступа:**

| Таблица | SELECT | INSERT | UPDATE | DELETE |
|---------|--------|--------|--------|--------|
| `profiles` | Все | При регистрации | Только свой профиль | Только свой профиль |
| `sketches` | Только approved | Авторизованные | Владелец | Владелец |
| `sketch_likes` | Все | Авторизованные | Владелец | Владелец |
| `sketch_comments` | Все | Авторизованные | Владелец | Владелец |

### Проверка работы авторизации

**Рабочий процесс:**

1. **Регистрация пользователя:**
   - Пользователь вводит email, пароль и display name
   - Вызывается `supabase.auth.signUp()`
   - Создаётся пользователь в `auth.users`
   - Триггер `on_auth_user_created` автоматически создаёт профиль в `profiles`
   - Профиль загружается в приложение через `loadProfile()`

2. **Вход пользователя:**
   - Пользователь вводит email и пароль
   - Вызывается `supabase.auth.signInWithPassword()`
   - Загружается профиль через `loadProfile()`
   - Обновляется состояние в `useAuth`

3. **Загрузка аватара:**
   - Пользователь выбирает файл изображения
   - Файл загружается в Storage: `user-content/avatars/{user-id}-{timestamp}.ext`
   - Получается публичная ссылка через `getPublicUrl()`
   - Профиль обновляется с новой ссылкой на аватар

4. **Обновление профиля:**
   - Пользователь редактирует display name и bio
   - Вызывается `updateProfile()` с новыми данными
   - Профиль обновляется в БД и в состоянии приложения

5. **Автосохранение сессии:**
   - При загрузке приложения проверяется сессия через `getSession()`
   - Подписка на изменения через `onAuthStateChange()`
   - При изменении состояния загружается профиль пользователя

**Статус:** ✅ Все компоненты работают корректно

```
1. Пользователь создаёт скетч → status: 'pending'
2. Скетч появляется в AdminDashboard
3. Модератор просматривает код и превью
4. Одобрить → status: 'approved' (появляется в галерее)
   Отклонить → status: 'rejected' + причина (уведомление автору)
```
