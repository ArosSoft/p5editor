## 🏗️ Архитектура приложения

### Основные компоненты

| Компонент | Описание | Технологии |
|-----------|----------|------------|
| **App.vue** | Главный контейнер, роутинг через `RouterView` | Vue 3, Vue Router |
| **EditorPage.vue** | Страница редактора: код, холст, консоль, AI | Vue 3, CodeMirror 6 |
| **ExplorePage.vue** | Галерея скетчей сообщества с поиском и фильтрами | Vue 3, Mock API |
| **SharePage.vue** | Страница публикации скетча | Vue 3 |
| **SketchDetailPage.vue** | Детальный просмотр скетча | Vue 3 |
| **CodeEditor.vue** | Редактор кода с подсветкой синтаксиса | CodeMirror 6, vue-codemirror6 |
| **P5Canvas.vue** | Изолированный iframe для выполнения p5.js скетчей | Iframe sandbox, Blob URL |
| **ConsoleOutput.vue** | Консоль для вывода сообщений и ошибок | Vue 3 |
| **ExamplesPanel.vue** | Галерея примеров с пошаговыми уроками | Vue 3, marked (markdown) |
| **AIChat.vue** | Чат с Deepseek AI для помощи с кодом | Deepseek API, Vue 3 |

### Маршруты (Vue Router)

| Путь | Страница | Описание |
|------|----------|----------|
| `/` | EditorPage | Редактор кода (основная страница) |
| `/explore` | ExplorePage | Галерея скетчей сообщества |
| `/sketch/:id` | SketchDetailPage | Детали скетча |
| `/share` | SharePage | Публикация скетча |

### Поток данных

```
Router (Vue Router)
    │
    ├── EditorPage.vue
    │   ├── CodeEditor.vue ←→ code (v-model)
    │   ├── P5Canvas.vue → addMessage(), mouseMove
    │   ├── ConsoleOutput.vue → messages
    │   ├── ExamplesPanel.vue → loadExample(code)
    │   └── AIChat.vue → suggestCode(code)
    │
    ├── ExplorePage.vue
    │   └── mock/sketches.ts → search, filter, sort
    │
    ├── SharePage.vue
    │   └── Форма публикации → mock/real API
    │
    └── SketchDetailPage.vue
        └── mock/sketches.ts → getSketchById()
```

### Управление состоянием

- **Локальное состояние:** Каждый компонент управляет своим состоянием через `ref()`, `reactive()`
- **Двустороннее связывание:** `v-model` для редактора кода
- **События:** `emit()` для коммуникации между компонентами
- **localStorage:** Сохранение состояния в ExamplesPanel (выбранный пример, шаг, вид)
- **Mock API:** Функции в `src/mock/sketches.ts` для работы со скетчами сообщества

```typescript
interface Sketch {
```

- `getPopularSketches(limit)` — популярные скетчи
- `getNewSketches(limit)` — новые скетчи
- `getAllTags()` — все теги
- `getAllCategories()` — все категории

### 4.3 Модерация скетчей

#### Админ-панель

**Файл:** `src/views/AdminDashboard.vue`

**Функционал:**
- Список скетчей на модерации (status: pending)
- Предпросмотр скетча (код + превью)
- Кнопки «Одобрить» / «Отклонить»
- Причина отклонения (textarea)
- Фильтры (статус, автор, дата)

#### Роли пользователей

```typescript
type UserRole = 'user' | 'moderator' | 'admin'

interface User {
  id: string
  email: string
  displayName: string
  role: UserRole
  avatar?: string
}
```

#### Статусы скетчей

```typescript
type SketchStatus = 'pending' | 'approved' | 'rejected' | 'draft'
```

#### Workflow модерации

```
1. Пользователь отправляет скетч → status: 'pending'
2. Модератор видит в AdminDashboard
3. Просматривает код и превью
4. Одобрить → status: 'approved' (появляется в галерее)
   Отклонить → status: 'rejected' + причина (уведомление автору)
```

### 4.4 Безопасность

#### Правила для опубликованных скетчей

- **Запрещено:**
  - XSS атаки (script injection)
  - Доступ к внешним API с чувствительными данными
  - Майнинг криптовалюты
  - Вредоносный код

#### Sandbox для iframe

```typescript
// P5Canvas.vue
sandbox="allow-scripts allow-same-origin allow-popups"
```

**Дополнительные меры:**
- Ограничение размера кода (max 50KB)
- Блокировка `eval()`, `Function()` в пользовательском коде
- Content Security Policy (CSP) заголовки

## 📦 Дополнительные рекомендации

### Хранение данных

Для production рекомендуется:
- **Скетчи:** База данных (PostgreSQL/MongoDB)
- **Изображения:** Cloud storage (Firebase Storage, AWS S3)
- **Код:** Хранить в БД или как отдельные файлы

### Backend опции

1. **Firebase** — быстро, бесплатно для старта
   - Firebase Auth (авторизация)
   - Firestore (база данных)
   - Firebase Storage (файлы)

2. **Supabase** — PostgreSQL + auth + storage
   - Встроенная авторизация
   - Real-time подписки
   - REST API из коробки

3. **Custom Node.js** — полный контроль, сложнее в поддержке
   - Express + JWT
   - PostgreSQL/MongoDB
   - Multer (загрузка файлов)

### CI/CD

- GitHub Actions для автотестов
- Автоматический деплой на Vercel/Netlify
- Preview для pull request'ов
