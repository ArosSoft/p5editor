## 🏗️ Архитектура приложения

### Основные компоненты

| Компонент | Описание | Технологии |
|-----------|----------|------------|
| **App.vue** | Главный контейнер, управление состоянием, панель инструментов, ресайз панелей | Vue 3 Composition API |
| **CodeEditor.vue** | Редактор кода с подсветкой синтаксиса | CodeMirror 6, vue-codemirror6 |
| **P5Canvas.vue** | Изолированный iframe для выполнения p5.js скетчей | Iframe sandbox, Blob URL |
| **ConsoleOutput.vue** | Консоль для вывода сообщений и ошибок | Vue 3 |
| **ExamplesPanel.vue** | Галерея примеров с пошаговыми уроками | Vue 3, marked (markdown) |
| **AIChat.vue** | Чат с Deepseek AI для помощи с кодом | Deepseek API, Vue 3 |

### Поток данных

```
App.vue (состояние: code, theme, messages)
    │
    ├── CodeEditor.vue ←→ code (v-model)
    ├── P5Canvas.vue → addMessage(), mouseMove
    ├── ConsoleOutput.vue → messages
    ├── ExamplesPanel.vue → loadExample(code)
    └── AIChat.vue → suggestCode(code)
```

### Управление состоянием

- **Локальное состояние:** Каждый компонент управляет своим состоянием через `ref()`, `reactive()`
- **Двустороннее связывание:** `v-model` для редактора кода
- **События:** `emit()` для коммуникации между компонентами
- **localStorage:** Сохранение состояния в ExamplesPanel (выбранный пример, шаг, вид)

## 1. Страница «Исследуй» — Галерея скетчей сообщества

### Цель
Создание отдельной страницы для просмотра, поиска и фильтрации скетчей, опубликованных сообществом пользователей.

### Задачи

#### 1.1 Настройка маршрутизации
```bash
npm install vue-router@4
```

**Файлы:**
- `src/router/index.ts` — конфигурация роутера
- `src/main.ts` — подключение роутера

**Маршруты:**
```typescript
{
  path: '/', component: EditorPage
  path: '/explore', component: ExplorePage
  path: '/sketch/:id', component: SketchDetailPage
  path: '/share', component: SharePage
}
```

#### 1.2 Создание компонента ExplorePage

**Файл:** `src/views/ExplorePage.vue`

**Функционал:**
- Сетка карточек скетчей (изображение, название, автор, лайки)
- Поиск по названию и тегам
- Фильтры (темы, сложность, популярные, новые)
- Сортировка (дата, популярность, название)
- Пагинация или infinite scroll

**Структура карточки:**
```vue
<div class="sketch-card">
  <img :src="sketch.thumbnail" class="thumbnail" />
  <div class="sketch-info">
    <h3>{{ sketch.title }}</h3>
    <p class="author">by {{ sketch.author }}</p>
    <div class="tags">
      <span v-for="tag in sketch.tags">{{ tag }}</span>
    </div>
    <div class="stats">
      <span>❤️ {{ sketch.likes }}</span>
      <span>👁️ {{ sketch.views }}</span>
    </div>
  </div>
</div>
```

#### 1.3 API для получения скетчей

**Эндпоинты (требуют backend):**
- `GET /api/sketches` — список всех скетчей
- `GET /api/sketches/:id` — детали скетча
- `GET /api/sketches/search?q=...` — поиск
- `GET /api/sketches/tags` — список тегов

**Создание thumbnail:**
```typescript
function captureThumbnail() {
  const canvas = document.querySelector('canvas')
  if (canvas) {
    const thumbnail = canvas.toDataURL('image/png')
    return thumbnail
  }
}
```

## 4. Авторизация и модерация

### 4.1 Установка компонентов авторизации

#### Вариант A: Firebase Authentication (рекомендуется)

```bash
npm install firebase
```

**Преимущества:**
- Готовая инфраструктура
- Email/пароль, Google, GitHub провайдеры
- Бесплатный тариф для небольших проектов

**Файлы:**
- `src/firebase/config.ts` — конфигурация Firebase
- `src/composables/useAuth.ts` — хуки для авторизации
- `src/components/AuthModal.vue` — модальное окно входа/регистрации

**Пример конфигурации:**
```typescript
// src/firebase/config.ts
import { initializeApp } from 'firebase/app'
import { getAuth } from 'firebase/auth'

const firebaseConfig = {
  apiKey: "YOUR_API_KEY",
  authDomain: "your-app.firebaseapp.com",
  projectId: "your-app",
  // ...
}

const app = initializeApp(firebaseConfig)
export const auth = getAuth(app)
```

#### Вариант B: Supabase Auth

```bash
npm install @supabase/supabase-js
```

**Преимущества:**
- PostgreSQL база в комплекте
- Готовые API для данных
- Встроенная авторизация

#### Вариант C: Custom Backend (Node.js + Express)

```bash
npm install express jsonwebtoken bcryptjs cors
```

**Файлы backend:**
- `server/index.js` — Express сервер
- `server/routes/auth.js` — маршруты авторизации
- `server/middleware/auth.js` — проверка JWT токенов

### 4.2 Компоненты авторизации

#### AuthModal.vue

**Функционал:**
- Вкладка входа (email + пароль)
- Вкладка регистрации (email + пароль + подтверждение)
- Кнопки соцсетей (Google, GitHub)
- Сброс пароля

#### UserProfile.vue

**Файл:** `src/components/UserProfile.vue`

**Данные:**
- Аватар (загрузка/генерация)
- Display name
- Bio
- Статистика (скетчи, лайки, просмотры)

#### Composable useAuth

**Файл:** `src/composables/useAuth.ts`

```typescript
export function useAuth() {
  const user = ref<User | null>(null)
  const loading = ref(false)

  async function login(email: string, password: string) { ... }
  async function register(email: string, password: string) { ... }
  async function logout() { ... }
  async function resetPassword(email: string) { ... }

  return { user, loading, login, register, logout, resetPassword }
}
```

### 4.3 Модерация скетчей

#### Админ-панель

**Файл:** `src/views/AdminDashboard.vue`

**Функционал:**
- Список скетчей на модерации (status: pending)
- Предпросмотр скетча (код + превью)
- Кнопки «Одобрить» / «Отклонить»
- Причина отклонения (textarea)
- Фильтры (статус, автор, дата)

#### Роли пользователей

```typescript
type UserRole = 'user' | 'moderator' | 'admin'

interface User {
  id: string
  email: string
  displayName: string
  role: UserRole
  avatar?: string
}
```

#### Статусы скетчей

```typescript
type SketchStatus = 'pending' | 'approved' | 'rejected' | 'draft'
```

#### Workflow модерации

```
1. Пользователь отправляет скетч → status: 'pending'
2. Модератор видит в AdminDashboard
3. Просматривает код и превью
4. Одобрить → status: 'approved' (появляется в галерее)
   Отклонить → status: 'rejected' + причина (уведомление автору)
```

### 4.4 Безопасность

#### Правила для опубликованных скетчей

- **Запрещено:**
  - XSS атаки (script injection)
  - Доступ к внешним API с чувствительными данными
  - Майнинг криптовалюты
  - Вредоносный код

#### Sandbox для iframe

```typescript
// P5Canvas.vue
sandbox="allow-scripts allow-same-origin allow-popups"
```

**Дополнительные меры:**
- Ограничение размера кода (max 50KB)
- Блокировка `eval()`, `Function()` в пользовательском коде
- Content Security Policy (CSP) заголовки
